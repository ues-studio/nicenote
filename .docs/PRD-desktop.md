# NiceNote 桌面版产品需求文档 (PRD)

> 角色：UX 专家 + 产品经理
> 技术栈：Wails3 (Go + React 19)
> 核心理念：**文件夹即笔记本，.md 文件即笔记**

---

## 一、产品定位与目标

### 1.1 产品愿景

**"你的文件就是你的笔记"** —— 打开任意文件夹，里面的 Markdown 文件就是你的笔记库。无私有格式，无数据绑定，用你熟悉的工具（Git、iCloud、Obsidian、VSCode）管理同一份数据。

### 1.2 核心数据模型

```
文件系统（用户选择的文件夹）
  笔记本/
    笔记1.md
    笔记2.md
    子文件夹/
      笔记3.md
```

- **笔记 = .md 文件**（Markdown 格式，人类可读）
- **文件夹 = 分类/项目**（嵌套结构天然支持层级）
- **无数据库依赖**：笔记数据 100% 在用户控制的文件中
- **与其他工具共存**：同一文件夹可同时用 VSCode、Obsidian、Git 管理

### 1.3 核心差异化价值

| 对比维度    | NiceNote Desktop         | Obsidian           | Notion   |
| ----------- | ------------------------ | ------------------ | -------- |
| 数据格式    | 标准 .md 文件            | 标准 .md 文件      | 私有格式 |
| 编辑体验    | WYSIWYG + MD             | 纯 MD / 双视图     | 块编辑   |
| 学习成本    | 极低（像记事本一样打开） | 中（插件生态复杂） | 高       |
| 与 Git 协作 | 天然支持                 | 天然支持           | ❌       |
| 多工具并用  | ✅                       | ✅                 | ❌       |

### 1.4 目标用户

- **技术写作者**：README、文档、博客草稿全用 Markdown
- **知识工作者**：用文件夹管理项目笔记，在意数据主权
- **从 Typora 迁移的用户**：熟悉 Markdown，想要更好的文件管理界面

## 二、核心工作流

### 2.1 打开文件夹（类 VSCode）

```
启动 NiceNote
    ↓
欢迎页：[打开文件夹] 或 [最近打开的文件夹]
    ↓
系统文件选择器 → 选择一个目录
    ↓
应用读取该目录下所有 .md 文件，展示在笔记列表
    ↓
开始工作
```

- 支持记住最近打开的文件夹（最多 10 个），快速切换
- 支持 **拖拽文件夹** 到应用窗口直接打开
- 支持命令行参数：`nicenote /path/to/folder`

### 2.2 文件保存策略

- 每次编辑内容变更，**800ms 防抖后直接写入对应 .md 文件**
- 标题修改 = 重命名文件（需用户确认或自动处理冲突）
- 新建笔记 = 在当前文件夹创建新 .md 文件
- 删除笔记 = 移入系统回收站（`os.Trash`），非永久删除

### 2.3 文件变更监听（File Watcher）

- 监听打开文件夹的文件系统变更（Go `fsnotify`）
- **外部程序修改** .md 文件时，自动刷新编辑器内容（提示用户或静默刷新）
- 新文件/删除文件自动更新笔记列表
- 编辑冲突处理：若当前正在编辑时文件被外部修改，提示用户选择保留哪个版本

## 三、信息架构（IA）

### 3.1 主界面布局（三栏）

```
┌──────┬───────────────────┬────────────────────────────────────┐
│ 导航 │    笔记列表         │            编辑区                   │
│ 64px │   260~400px       │           fill                      │
│      │                   │                                     │
│  📝  │  [搜索框]          │  [自定义标题栏：拖拽区 + 窗口控制]   │
│  📁  │  ─────────        │  ─────────────────────────────────  │
│  🏷  │  笔记卡片 1        │  [笔记标题（=文件名）]               │
│  ⭐  │  笔记卡片 2        │  [标签行 + 更新时间]                 │
│  ...  │  笔记卡片 3       │  ─────────────────────────────────  │
│      │  ...              │  [工具栏]                            │
│  ⚙️  │                   │  [Tiptap 编辑区]                    │
└──────┴───────────────────┴────────────────────────────────────┘
```

### 3.2 左侧导航视图

| 图标 | 视图     | 数据来源                               |
| ---- | -------- | -------------------------------------- |
| 📝   | 所有笔记 | 当前文件夹递归扫描所有 .md 文件        |
| 📁   | 文件夹树 | 文件系统目录树（只显示含 .md 的目录）  |
| 🏷   | 标签     | 解析笔记 frontmatter 中的 `tags:` 字段 |
| ⭐   | 收藏     | 存于本地 metadata 缓存（见 3.4）       |
| 🕐   | 最近修改 | 按文件 mtime 排序                      |
| ⚙️   | 设置     | 底部固定                               |

### 3.3 Frontmatter 规范（Markdown 元数据）

笔记元数据通过标准 YAML frontmatter 存储在 .md 文件中：

```yaml
---
tags: [工作, 想法]
created: 2025-01-15T10:00:00
---
# 笔记正文从这里开始
```

- `tags`：标签列表（字符串数组）
- `created`：创建时间（若文件已有则保留，否则写入文件创建时间）
- 与 Obsidian、Jekyll、Hugo 等工具兼容

### 3.4 元数据缓存（SQLite，仅缓存用途）

```
~/.config/NiceNote/cache.db（或 App Support）
  file_index: 文件路径 → mtime + 摘要缓存（避免每次读全文）
  favorites: 收藏的文件路径列表
  recent_folders: 最近打开的文件夹路径列表
  settings: 用户偏好（主题、语言、窗口大小等）
  tag_colors: 标签名 → 颜色映射（全局共享）
```

**关键原则**：SQLite 只存可再生的缓存数据，**删除 cache.db 不丢失任何笔记**。

## 四、核心功能需求（按优先级）

### P0 — MVP

#### 4.1 文件夹管理

- 打开文件夹（系统 File Picker）
- 记住最近打开的文件夹（最多 10 个）
- 欢迎页展示最近记录（类 VSCode Welcome）
- 切换文件夹
- 在访达/文件管理器中显示当前文件夹（"Reveal in Finder/Explorer"）

#### 4.2 笔记列表

- 递归扫描文件夹中的所有 .md 文件
- 展示：文件名（去掉 .md 后缀）作为标题 + 内容摘要（前 100 字）+ mtime
- 按最近修改时间排序（默认）
- 右键菜单：在 Finder 中显示、重命名、移动到子文件夹、删除

#### 4.3 笔记编辑

- 复用 `packages/editor` 的 Tiptap v3 编辑器（WYSIWYG）
- 内容变更 800ms 防抖后写入 .md 文件
- 标题编辑 → 对应修改文件名（1.5s 延迟重命名，避免输入中频繁重命名）
- 源码模式：直接查看/编辑原始 Markdown

#### 4.4 文件变更监听

- `fsnotify` 监听打开目录的所有文件变化
- 外部修改：若当前未处于编辑状态 → 静默刷新
- 外部修改：若正在编辑 → 提示用户（保留我的版本 / 使用文件版本）
- 新增文件 → 自动出现在列表；删除文件 → 自动从列表移除

#### 4.5 全文搜索

- `Cmd/Ctrl+K` 唤起搜索弹窗
- Go 后端实时扫描 + 结果缓存（基于 mtime 增量更新）
- 搜索标题 + 内容，高亮匹配片段
- 支持中文（Unicode 字符级分词）

#### 4.6 深浅主题

- 跟随系统（默认）/ 手动覆盖
- 设置存 SQLite cache

---

### P1 — V1.1

#### 4.7 标签系统（基于 Frontmatter）

- 解析所有 .md 文件的 `tags:` frontmatter 字段
- 左导航"标签视图"：展示所有标签 + 数量（汇聚所有文件的标签）
- 在编辑器中添加/删除标签 → 写回文件 frontmatter
- 按标签过滤笔记列表
- 标签颜色存于全局 SQLite cache

#### 4.8 文件夹树视图

- 左导航展示文件系统目录树（仅含 .md 文件的文件夹）
- 新建子文件夹
- 拖拽笔记移动到不同文件夹（= 移动文件）
- 文件夹折叠/展开状态持久化

#### 4.9 快速捕获（全局快捷键）

- `Cmd+Shift+N` / `Ctrl+Shift+N` 全局唤起小浮窗
- 输入标题和内容后保存为新 .md 文件（默认保存到当前打开文件夹根目录）
- 浮窗 Esc 或失焦关闭

#### 4.10 系统托盘

- macOS 菜单栏 / Windows 系统托盘常驻
- 右键菜单：显示主窗口、新建笔记、快速捕获、退出

#### 4.11 导入 / 导出

- **导入**：将 .md 文件拖入窗口 → 复制到当前文件夹
- **导出当前笔记**：系统 Save Dialog 另存为 .md（内容本就是 .md，直接复制）
- **导出为 PDF**：调用系统打印为 PDF

---

### P2 — V1.2+

#### 4.12 多文件夹/多工作区

- 同时打开多个文件夹（类 VSCode 多工作区）
- 或左侧收藏常用文件夹路径快速切换

#### 4.13 图片处理

- 粘贴/拖拽图片 → 自动保存为 `.md` 文件同级的 `assets/` 文件夹
- Markdown 引用：`![](./assets/image.png)`
- 与 Obsidian 的图片存放惯例兼容

#### 4.14 收藏（书签）

- 收藏某篇笔记（存于 SQLite cache）
- 左导航"收藏"视图

#### 4.15 多窗口

- 在新窗口打开某篇笔记（`Cmd+Shift+Enter`）
- 两个窗口同时编辑同一文件时的冲突处理

## 七、非功能需求

### 7.1 数据安全

- 笔记完全在用户文件系统中，应用不上传任何数据
- 写入前先写临时文件，成功后原子替换，防止写入中断导致文件损坏

### 7.2 与其他工具的兼容性

- Frontmatter 格式与 **Obsidian / Jekyll / Hugo / Hexo** 兼容
- 文件命名使用 UTF-8，支持中文文件名
- 目录保持干净，不创建私有元数据文件夹（如 `.nicenote/`）

### 7.3 可访问性

- 完整键盘操作
- 支持系统辅助功能（VoiceOver / Narrator）
- 高对比度模式
- 中英文双语（复用 `apps/web` i18n 资源）

## 九、已确认的设计决策

1. **文件名即标题**：
   - 编辑器顶部标题输入框 = 文件名（去掉 .md 后缀）
   - 修改标题 → 1.5s 防抖后重命名文件（避免输入中频繁重命名）
   - 新建笔记默认文件名：`YYYY-MM-DD HH:mm.md`，用户修改标题后自动重命名

2. **元数据存储**：
   - 标签颜色等配置存于 `SQLite cache`（全局，`App Support/NiceNote/cache.db`）
   - 标签与笔记的关联仍通过笔记 frontmatter `tags:` 字段存储（保持文件可移植性）
   - SQLite 仅存标签颜色、收藏列表、最近文件夹、用户偏好

3. **单文件夹模式（MVP）**：
   - 同一时间只打开一个文件夹
   - 左侧收藏/最近列表支持快速切换文件夹
   - 多文件夹聚合视图推迟到 V1.2+
